Faux borders & columns

This page is demonstrating a few techniques, with advantages and disadvantages, for creating columns & bordered elements without breaking a layout grid. The techniques outlined should work in all the common frameworks.

Example one

This first example uses a background image that's ridiculously wide and has 2 'columns'. The divide between the columns is calculated based on the width of the page, and is positioned accordingly.

I don't do a very good job of detailing this, so I'll leave it to the pros. This technique was originally crafted by Dan Cederholm.

The CSS

#demo-one {
	background: url(background.png) repeat-y 74% top;
}

Advantages

  • Minimal markup & CSS
  • Fully compatible
  • Flexible
  • Spans the full height of the container

Disadvantages

  • Scales strangely at very narrow widths
  • To change colors, you have to create a new image
  • Fails if the user just wants to have one column without changing classes etc

Works in: all the things.

Example two

This example is for borders, but can be used for columns similar to example one. By using 2 box-shadow rules, you can emulate a border divide between the columns without using padding or real borders, which would break the grid.

The CSS

#demo-two .col_3 {
	-webkit-box-shadow: -23px 0 0 #fff, -24px 0 0 #bbb;
	-moz-box-shadow: -23px 0 0 #fff, -24px 0 0 #bbb;
	-o-box-shadow: -23px 0 0 #fff, -24px 0 0 #bbb;
	box-shadow: -23px 0 0 #fff, -24px 0 0 #bbb;
}

Advantages

  • No images
  • Colors are easily adjusted

Disadvantages

  • Not fully compatible
  • Doesn't span the height of the container
  • The primary box-shadow must always match the background color of the page. Not a fun task to maintain
  • Breaks on non-fixed widths. You have to set the shadow distances in an absolute unit - in fact, the column will overlap the content in smaller sizes on this example

Works in: browsers with support for box-shadow.

Example three

The final (and probably my favorite) example uses the :before pseudo element to create a border. It just makes a 1px wide element, pulled back by 10% of the column width, thne given a z-index of -1 to prevent overlap.

The CSS

#demo-three {
	position: relative;
}

#demo-three:before {
	content: ".";
	height: 100%;
	display: block;
	position: absolute;
	width: 1px;
	background: #bbb;
	text-indent: -9999px;
	left: 76.5%;
	top: 0;
	z-index: -1;
}

Advantages

  • Great browser support
  • Colors are easily adjusted
  • Flexible - supports percentage based values, so it plays nicely with the grid
  • Transparent background means no faffing around with matching up colors

Disadvantages

  • Not fully compatible
  • Doesn't span the height of the container
  • Can be a little funny at narrow widths, but still pretty good

Works in: browsers with support for :before - which is all of them, apart from IE7 and below.
